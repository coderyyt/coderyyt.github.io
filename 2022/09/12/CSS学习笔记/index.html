    CSS属性 - 文本

    text-decoration装饰线

    • none: 无任何装饰线（可以去除a元素默认的下划线）
    • underline: 下划线
    • overline: 上划线
    • line-through: 中划线（删除线）

    text-transform(了解)

    • capitalize：首字母大写
    • uppercase：全大写
    • lowercase：全小写

    text-indent(了解)

    • 2em：首行缩进

    text-align(重要)

    center：正中间显示

    • 文字居中
    • 图片居中
    • div元素的居中(W3C inline-level)
      • 特性 - 针对行内级（inline-level）对齐方式，对块级元素不起作用（如：div）
      • 解决方法 - 改为行内级(需要text-align)或margin0 auto(不需要text-align)

    justify：两端对齐（了解）

    • 特性：边缘两端对齐（默认最右边会有间隔，最后一行不生效）
    • 若最后一行需要两端对齐，设置：text-align-last: justify;

    letter-word-spacing(了解)

    • letter-spacing - 单位为px，设置文本间的空隙
    • word-spacing - 单位为px，设置文本空格的间距

    CSS属性 - 字体

    font-size

    • 20px;
    • 2em; em -> 父元素的字体尺寸(1em = 100%)
    • 200% 百分比 -> 父元素的字体尺寸

    font-family

    • 原理: 读取操作系统的字体
      • 网络字体 - 通过**@font-face**指定下载字体
    • 1个或者多个值

    font-weight

    常见取值：100 | 200 | 300 | … | 900

    normal： 400

    bold：700

    font-style

    设置字体斜体

    normal默认

    italic斜体：字体本身支持斜体时, 显示斜体

    oblique倾斜：让文本进行倾斜

    font-varient

    • 小写字母以大写显示, 但是高度保持小写的高度

    line-height(重要)

    一行文字所占据的高度，有文本居中的作用

    为什么文本需要行高，方便阅读

    基线：英语草稿纸x底部的线

    • 两行文字基线(baseline)之间的距离
    • 一行文本 -> line-height
    • 行高 - 文本高度 = 行距

    font缩写属性

    • font-size/line-height font-family;固定写法，不能调换顺序，line-height可以省略

    一般用来给元素作通用性设置，如内边距，外边距，重置一些内容

    • 元素 div
    • 类 .class

    • 名称最好不要是标签名，阅读性差

    继承-层叠-元素类型

    CSS属性继承

    • 继承: 如果我们给某一个元素设置了某个CSS属性, 而该属性具有继承性, 那么该元素的所有子元素会默认继承该属性.
    • 有哪些继承属性? 不需要记
      • 一般和文本/字体相关的很多属性都具备继承;
      • xmind
      • 多查文档

    CSS属性层叠

    • 一个CSS属性可以多次设置:
      • 判断一: 权重, 优先级;
      • 判断二: 先后顺序;
    • 常见的选择器:
      • !important -> 10000
      • 内联 -> 1000
      • id -> 100
      • 类/伪类/属性 -> 10
      • 元素 -> 1
      • 统配 -> 0

    元素的类型

    • div -> 块级元素
    • span -> 行内级元素(行内块级元素、行内非替换元素)
    • 有本质的区别 -> display: block
    • span -> 块级
    • div -> 行内级

    display:

    • block
      • 独占一行(父元素)
      • 可以设置宽度和高度
      • 默认高度是内容的高度
    • inline-block
      • 和其他行内级元素在同一行显示
      • 可以设置宽度和高度
      • 默认宽高由内容决定
    • inline
      • 和其他行内级元素在同一行显示
      • 不能设置宽度和高度
      • 宽高由内容决定

    元素的隐藏方法

    • display: none 不占据空间(页面)
    • visibility: hidden(元素不可见) 默认可见(visibile) 占据空间
    • rgba(a的值为0~1) -> alpha
    • bgc的transparent相当于rgba(0,0,0,0)
    • opacity -> 设置透明度，值为0~1
      • 所有的子元素都会跟着一起设置

    overflow

    内容溢出边框时，以某种方式显示

    • visible 默认，内容不会被修剪，会呈现在元素框之外
    • hidden 内容被修剪，超出部分不可见
    • scroll 内容被修剪，浏览器会显示滚动条以便查看其余的内容(右滚动条、下滚动条)
    • auto 如果内容被修剪，浏览器会显示滚动条以便查看其余的内容(右滚动条)

    标准流布局

    一般情况下，元素都是按照标准流(normal flow)进行排布

    从左到右、从上往下按顺序摆放

    一般情况下，互相之间不存在层叠现象

    在标准流中，使用margin、padding对元素进行定位

    其中margin还可以设置负数

    缺点：

    • 设置一个元素的margin或padding，会影响到标准流中其他元素定位效果
    • 不便于实现元素层叠效果

    希望一个元素可以跳出标准流，单独的对某个元素进行定位

    • 通过position元素来进行设置

    定位

    从正常的文档流布局中取出元素，使它们具有不同的行为：

    • 将元素放在另一个元素上面（层叠）
    • 始终保存在浏览器视窗内的同一位置

    定位在开发中非常常见

    利用position可以对元素进行定位，常用取值有5个：

    默认值是static：静态定位

    使用下面值，让元素变成定位元素

    relative：相对定位

    absolute：绝对定位

    fixed：固定定位

    sticky：粘性定位

    静态定位static

    不脱标，通过left、right、top、bottom没有任何作用

    相对定位relative

    不脱标，可以通过left、right、top、bottom进行定位

    定位参照对象是元素自己原来的位置，原来的位置保留占位

    相对定位的应用场景

    不影响其他元素位置的前提下，对自身元素位置进行微调

    子绝父相的误区（子绝 -> 祖先元素中的定位元素）

    绝对定位是相对于最近一个定位元素

    固定定位fixed

    脱标，通过left、right、top、bottom进行定位

    定位参照对象是浏览器视口

    有画布滚动时，固定不动

    视口：浏览器可视区域

    画布：渲染文档区域，文档内容超过视口范围，通过滚动查看

    宽高对比：画布 >= 视口

    绝对定位absolute

    脱标，通过left、right、top、bottom进行定位

    定位参照对象是最邻近的定位祖先元素

    如果找不到这样的祖先元素，参照对象是视口

    定位元素：

    position值不为static的元素(默认)

    position值为：relative相对、absolute绝对、fixed固定

    在绝大数情况下，子元素的绝对定位都是相对于父元素进行定位

    如果希望子元素相对于父元素进行定位，又不希望父元素脱标，常见解决方案：

    父元素为**relative(**父元素为定位元素，且不托标)

    子元素为absolute

    简称：“子绝父相”

    将position设置为absolute、fixed元素的特点一

    • 可以随意设置宽高
    • 宽高默认由内容决定
    • 不再受标准流约束
      • 不再严格按照从上到下、从左到右排布
      • 不再严格区分block、inline、inline-block的特性都会消失
    • 不再给父元素汇报宽高数据
    • 脱标元素内部默认还是按照标准流布局

    将position设置为absolute、fixed元素的特点二

    • absolute、fixed两个算是绝对定位(字面上)，absolute定位参照对象为最邻近的定位祖先元素(找不到这样的祖先元素，参照对象为视口)；而fixed定位参照对象为视口

    • 公式：

      • 定位参照对象宽度 = left + right + m-l + m-r + 绝对定位元素的实际占用宽度
      • 定位参照对象高度 = top + bottom + m-t + m-b + 绝对定位元素的实际占用高度

    • 绝对定位元素的宽高和定位参照对象一样，可以给绝对定位元素设置以下属性：

      • left：0、right：0、top：0、bottom：0、margin：0(无视块级属性)

    • 绝对定位元素在定位参照对象中居中显示，可以给绝对定位元素设置以下属性：

      • left：0、right：0、top：0、bottom：0、margin：auto
      • 另外，还得设置具体的宽高值(宽高小于定位参照对象的宽高)

    auto是什么？

    auto -> 交给浏览器处理

    width: auto;

    • 行内非替换元素 -> width: 包裹内容
    • 块级元素 -> width: 包含快内容
    • 绝对定位换元素 -> width: 包裹内容

    粘性定位sticky

    比其他定位值要新一些

    sticky可以看做是相对定位和固定（绝对）定位的结合体；

    被定位的元素表现得像相对定位一样，知道滚动到某个阈值点；

    当达到这个阈值点时，就会变成固定（绝对）定位；

    sticky是相对于最近的滚动祖先包含视口

    z-index

    用来设置定位元素的层叠顺序

    • 取值可以是正整数、负整数、0、
    • 比较原则：
      • 兄弟关系：
      • z-index越大，层叠在越上面
      • z-index相等，写在后面的那个元素层叠在上面
      • 不是兄弟关系：
      • 各自从元素自己以及祖先元素中，找出最邻近的2个定位元素进行比较
      • 而且这2个定位元素必须有设置z-index的具体数值

    浮动

    规则：

    • 元素浮动脱标
      • 定位元素会层叠浮动元素上
      • 层叠顺序：普通非定位元素 -> 浮动元素 -> 定位元素
    • 元素向左(右)浮动，浮动元素的左(右)边界不能超出包含快的左(右)边界
    • 浮动元素之间不能层叠
      • 左浮找左浮，右浮找右浮
      • 剩余空间不够显示浮动元素，浮动元素将向下移动，直到有充足空间为止
      • 浮动元素不能与行内块内容层叠，会被推出去
    • 行内及元素、inline-block元素浮动后，其顶部将于所在行的顶部对齐
