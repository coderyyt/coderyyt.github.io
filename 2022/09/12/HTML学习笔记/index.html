剖析元素结构

什么是元素

  • 开始标签
  • 结束标签
  • 内容
  • 属性
  • 元素的嵌套

head

  • 元数据(配置信息)
    • meta -> charset -> utf-8
    • title -> 标题

img元素

  • 两个属性

    • src

    • alt:

      • 作用一：图片加载失败，显示这段文本

  • 地址

    • 网络地址
    • 本地地址
      • 绝对路径
      • 相对路径

a元素

  • 两个属性:
    • href
    • target（目的）
      • _self 自己 默认值，当前窗口
      • _blank 空白 在新的窗口
      • _parent 在父窗口中（iframe元素）
      • _top 在顶层窗口（iframe元素）
  • 本页面的锚点链接
  • a元素除了可以跳转目标外，还可以是压缩包（下载该压缩包）、邮箱（mailto:邮箱地址）

iframe元素（理解）

  • 嵌套网页

  • 某些网页禁止嵌套(原理)

    • 请求头中被设置同源

  • 和a元素结合(target另外两个值)

    • _self 在边框内中打开
    • _blank 在新的窗口打开
    • _parent 在ifram多层嵌套中，打开当前位置的上一级
    • _top 在ifram多层嵌套中，打开顶层的位置 body

不常用元素

  • strong
  • i
  • code
  • br
  • …

全局属性

  • id 作为a元素的锚点，也可以通过css、js来选择和访问特定的元素
  • class 通过css、js来选择和访问特定的元素
  • style 元素添加内联样式
  • title 提示符

URL

  • URL的格式:
    • 协议://主机:端口/路径/文件名?查询#片段id
  • 和URI的区别:
    • URI:标志符
    • URL: 定位符(网络地址)

元素语义化

  • 用正确的元素去做正确的事情

理论上div可以做其他元素的样式，如h1标签，在开发中尽量不要这么做，不方便区分、维护

标签语义化的好处：

方便维护，减少开发者沟通成本，语音合成工具正确识别网页元素的用途，作正确的反应，有利于SEO

SEO优化

  • SEO原理
    • SEO，搜索引擎优化，提高搜索排名靠前
    • 元素语义化可以提高搜索排名靠前
  • 过程/流程

爬虫在索引区抓取网页 -> 将抓取到的网页存到临时库中处理 -> 临时库可以读取爬到的网页 -> 符合规则存储在索引去，不符合规则清理 -> 在索引区中进行分类、归档、排序，然后将结果反馈给用户
